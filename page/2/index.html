  • GROUP BY的分组依据不仅可以是一列，也可以是多个列，中间用逗号隔开；
  • 查询的列要么是GROUP BY子句中出现的，要么是聚合函数；
  • 分组的列如果有NULL，不管多少都将作为一个组；

过滤分组

在前面学习到的过滤数据的关键字为where，但是如果要对分组进行过滤却不能用where，这是因为where过滤先于分组。此时就需要用到另外一个关键词—-HAVING。HAVING是用来对分组之后的数据进行过滤的，例如查询user_info表中不止一个user的年龄有哪些：

WHERE和HAVING

  • where过滤行，而having过滤分组；
  • where支持的having都支持，having支持的where却不一定支持；
  • where在数据分组前进行过滤，而having在数据分组之后过滤；
  • SQL语句执行时where先于oderby执行，而having晚于oderby执行。

分组和排序

  • oder by根据指定的列进行排序输出所有原生行，而group by则根据指定的列分组输出顺序不确定的分组行；
  • oder by后面可以跟任意列，而group by后面则只能跟选择列或表达式；
  • oder by不一定需要使用，而如果同时查询列和聚集函数时则必须使用。

数据库、SQL与MySQL简介

发表于 2020-08-12 更新于 2020-08-14 分类于 MySQL ， study

数据库

  • 数据库(database)：存储有组织有结构的数据的文件的容器。

  • 表(table)：保存有组织有结构的数据的文件，表名在本数据库中是唯一的。

  • 模式(schema)：关于数据库和表的布局及特性的信息。

  • 列(column)：表中的一个字段。

  • 数据类型(datatype)：规定了每个列所允许存放的数据的类型。

  • 行(row)：表中的一条记录。

  • 主键(primary key)：一列或一组列，作为一条记录的唯一标识。

  • SQL：Structured Query Language，结构化查询语言，与数据库通信的语言。

MySQL

DBMS(数据库管理系统)可分为两类：一类为基于共享文件系统的DBMS，另一类为基于客户机—服务器的DBMS。MySQL是基于客户机—服务器的DBMS，无论客户机与服务器在不在同一计算机上，他们都要进行通信。

MySQL的优点：开源免费、简单易用、执行速度快、可信赖。

MySQL用户权限管理

发表于 2020-08-10 更新于 2020-09-09 分类于 MySQL

为什么进行用户权限管理

用户所需要的访问权限要恰到好处，既不能多也不能少。良好的用户权限控制有助于避免用户的恶意操作与无意识错误而出现的严重后果。

如何进行用户管理

MySQL的用户账号和信息存储在MySQL的mysql数据库中。

如何进行权限管理

若为新增用户，则无任何权限，需要为其分配访问权限。

访问权限的层次

  • 整个服务器，使用GRANT ALL和REVOKE ALL；
  • 整个数据库，使用ON database.*；
  • 特定的表，使用ON database.table；
  • 特定的列，例：grant select(id, se, rank) on testdb.apache_log to dba@localhost;；
  • 特定的存储过程，例：grant create routine on testdb.* to developer@’192.168.0.%’;
